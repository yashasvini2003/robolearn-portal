1. Introduction to Robotics

A robot is a machine that can sense its environment, make decisions, and perform actions automatically. We see robots in:

  • Factories (assembly robots, welders)
  • Hospitals (surgical robots, delivery bots)
  • Homes (vacuum robots)
  • Space (Mars rovers)

Most robots include three key parts:

  1. Brain – a microcontroller or computer.
  2. Sensors – to detect light, distance, lines, etc.
  3. Actuators – motors and servos to move.

Teaching tip: Ask students to list where they’ve seen robots in daily life. It makes the topic more relatable.

2. Sensors & Motors

For a robot to react intelligently, it needs sensors and motors.

Common Sensors

  • IR sensor – detects lines or obstacles.
  • Ultrasonic sensor – measures distance.
  • Light sensor – senses brightness.

Motors

Robots usually use DC motors or servo motors to move wheels or arms.

Example idea: A line-following robot uses IR sensors to detect a black line on a white surface and adjusts its motors to stay on track.

Teaching tip: Show real sensors or pictures, and ask students what each could be used for.

3. Coding a Simple Robot

Robots follow instructions written in code. Here is a simplified example of blinking an LED on a microcontroller: 

// Blink an LED (Arduino-style pseudocode)
void setup() {
  pinMode(13, OUTPUT);   // Set pin 13 as an output
}

void loop() {
  digitalWrite(13, HIGH); // Turn LED on
  delay(1000);            // Wait 1 second
  digitalWrite(13, LOW);  // Turn LED off
  delay(1000);            // Wait 1 second
}

The robot repeats the loop() function forever. We can use similar logic to move motors or read sensors.

Teaching tip: Let students change the delay time and predict what happens before running the code.

4. Mini Project: Line-Following Robot

In this mini project, students build a robot that follows a black line on a white surface.

Hardware (example)

  • 1 × Microcontroller (e.g., Arduino UNO)
  • 2 × DC motors + motor driver
  • 2 × IR line sensors
  • Chassis, wheels, battery pack

Basic Logic

  1. Read left and right sensor values.
  2. If left sees black, turn slightly left.
  3. If right sees black, turn slightly right.
  4. If both see white, go straight.
// Pseudocode for line-following logic
void loop() {
  int left = readLeftSensor();
  int right = readRightSensor();

  if (left == BLACK && right == BLACK) {
    moveForward();
  } else if (left == BLACK && right == WHITE) {
    turnLeft();
  } else if (left == WHITE && right == BLACK) {
    turnRight();
  } else {
    stopMotors(); // Lost the line
  }
}

Teaching tip: Have students first draw the decision table on paper, then translate it to code.

Troubleshooting & FAQs

Common issues beginners face when working with robotics, plus simple explanations you can share with students.

My robot is not moving at all.

Check the battery connections, ensure the power switch is on, and verify that the motor driver is wired correctly. Also confirm that your code is uploading to the correct board/port.

Sensors always read the same value.

Make sure the sensors are correctly powered (VCC/GND), positioned close enough to the surface, and that your code is reading the correct pins. Try printing the sensor values to the serial monitor.

The line-following robot keeps losing the line.

Increase the contrast between the line and the background, slow down the robot, and re-calibrate the threshold values you use for black vs white.

Students are confused by the code.

Break the code into small blocks, explain each function, and relate it back to a real-world decision (IF this happens, THEN do that). Encourage students to predict behavior before running the code.
